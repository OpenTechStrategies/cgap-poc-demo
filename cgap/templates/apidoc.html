Application Programming Interface (API)

The PoC Demo API offers programmatic access to the sample data, using a standard REST-style interface. It is a preview of what a full, production service API would look like.

The API endpoints described below are live — you can try them out in your browser, or using a command-line tool like curl or wget, or from program code. For example, to receive the list of sample organizations (represented in JSON format) from the API in your browser, just visit the endpoint directly:

  https://cgap.opentechstrategies.com/poc-demo/api/organizations/

Or get the same information on the command line:

  $ curl https://cgap.opentechstrategies.com/poc-demo/api/organizations/

Please note that because this is a demo, there is currently no authentication or authorization defined for the API. A real instance would have authentication and access-control in front of the API, so that only authorized parties could access the data.

API Entities and Endpoints

  • Organization (entity)

    • GET /organizations/

      Returns a list of all organizations.

    • POST /organizations/

      Creates a new organization object.

    • GET /organizations/{EIN}

      Returns a single organization.

    • POST /organizations/{EIN}

      Updates an existing organization object.

  • Proposal (entity)

    • GET /proposals/

      Returns a list of all proposals.

      Can optionally accept a s query parameter which defines a search term.

    • POST /proposals/

      Creates a new proposal object.

    • GET /proposals/{ID}

      Returns a single proposal.

    • POST /proposals/{ID}

      Updates an existing proposal object.
