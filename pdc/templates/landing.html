PDC Common Data Platform — Proof-of-Concept

This proof-of-concept shows severals screens demonstrating how properly access-controlled shared data can help grantseekers and grantmakers do a better job of finding each other, and can save grantseekers time when applying for funding.

(Note: all the data here is fictitious sample data created for this demo site. No actual organization data is used.)

  1. GMS application screen (simulation)

    An application screen in a prospective PDC-enabled Grant Management System (GMS).

    When the applicant starts typing their organization's name or EIN, the field offers autofills based on existing data. Once the organization is identified, all the rest of the fields fill in automatically, sparing the applicant the need to re-enter data that they have probably entered too many times already.

    If the applicant changes the information in one of the fields — say, there is a new CEO — the GMS saves the new information and sends it back to the PDC Common Data Platform. There, the organization's record is (potentially) updated, in accordance with data governance and update rules.

    The applicant takes no extra steps to make any of this happen. The GMS just uses data the applicant has entered before, perhaps elsewhere. And by accepting updates and sending them back to the CDP, the GMS contributes to future time-saving for the applicant.

  2. Search screen: find organizations by activity

    Shows how a funder can search to find new organizations, based on those organizations' activities, missions, etc, etc.

    The search includes both organizations themselves and their associated proposals. For example, it matches organization names and mission statements, and also matches proposal names and descriptions.

  3. Automated access — API documentation

    Other systems can access the Common Data Platform programatically, with proper authorization. The API ("Application Programming Interface") documentation shows how.
